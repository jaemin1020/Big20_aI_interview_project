# [리포트] 9번~14번 인성 면접(Behavioral Interview) 기술 고도화 상세 보고

본 보고서는 AI 인터뷰 엔진의 9번~14번 인성 면접 단계에서 발생하는 **'직무 질문 편중 현상'**을 해결하고, **'회사의 인재상'**과 **'지원자의 과거 경험'**을 유기적으로 결합하기 위해 적용된 기술적 수정 사항을 상세히 기술합니다.

---

## 1. 개요 및 문제 정의

기존 시스템은 인성 면접 단계(9~14번)에서 이력서 RAG(Retrieval-Augmented Generation)를 생략하거나 기술 중심의 단순 검색을 수행하여 다음과 같은 한계가 있었습니다.
- **맥락 부재**: 질문이 너무 추상적여서 지원자의 실제 경험과 동떨어짐.
- **직무 몰입**: RAG로 검색된 기술 키워드(SQL, Python 등)에 AI가 매몰되어 인성 면접임에도 직무 역량을 다시 묻게 됨.
- **인재상 결여**: 회사가 중시하는 가치관과 지원자의 과거 행동 양식 간의 연결고리 부족.

---

## 2. 주요 수정 사항 (Technical Implementation)

### **[1] 역량별 맞춤형 RAG 키워드 매핑 (Specialized RAG)**
단순히 "경험"을 검색하는 대신, 각 인성 면접의 세부 목적(협업, 성장, 책임감)에 부합하는 키워드로 이력서를 정밀 검색하도록 로직을 재설계했습니다.

**[적용 코드]**
```python
# 9-14번 인성 면접: 각 역량(협업, 성장, 책임감)에 특화된 RAG 수행
s_name = next_stage.get('stage', '')
behavioral_keywords = {
    "communication": "협업 사례, 팀 프로젝트 중 갈등 조율, 팀워크 발휘, 소통 능력",
    "growth": "자기계발 노력, 새로운 기술 학습 태도, 실패 극복 및 성장 사례",
    "responsibility": "직업 윤리, 약속 이행, 정직함과 관련된 경험"
}
# 해당 단계에 맞는 쿼리 선택 (없으면 기본 가치관 경험 검색)
target_query = behavioral_keywords.get(s_name, "본인의 강점, 성취감, 도전적인 경험 사례")

logger.info(f"✨ Behavioral RAG ({s_name}): Searching for '{target_query}'")
rag_results = retrieve_context(target_query, resume_id=interview.resume_id, top_k=2)
```

### **[2] '배경 기반' 프롬프트 및 동적 제약 조건 (Global Constraint)**
검색된 이력서 데이터를 **'배경 상황'**으로만 쓰도록 제한하고, 인성 면접 단계에서는 기술 용어(코드, 설계 등) 사용을 원천 금지하는 동적 제약 조건을 도입했습니다.

**[적용 코드]**
```python
# 기술/인성 단계별 전역 제약 조건 분리
if is_narrative:
    global_constraint = "인성 단계입니다. **코드, 설계, 개발과 같은 직무 단어를 사용하지 말고**, 태도와 가치관을 인용하여 짧게 질문하십시오."

# 프롬프트 주입 예시
context_text = (
    f"이 단계는 {next_stage['display_name']}를 확인하는 인성 면접입니다.\n"
    f"기술력 검증보다는 **태도, 가치관, 조직 적응력**을 파악하는 데 집중하십시오.\n"
    f"아래 [지원자 정보]를 '배경'으로만 활용하십시오."
)
```

### **[3] 인성 면접 전용 전략 및 길이 축소 (Contextual UX)**
AI의 작업 지시어를 더 세밀하게 조정하여 문장 길이를 기존 대비 **30% 축소**하고, 불필요한 서술을 제거했습니다.

**[적용 코드]**
```python
elif s_name == 'responsibility':
    # 11번 가치관 질문: 기술 중심 인용 대신 가치 중심 전환 (전체 길이 30% 축소)
    mode_task_instruction = "자소서의 핵심 가치관을 파악하여 인재상과 연결하십시오. 서비스 설계나 코드 같은 기술 내용은 일절 언급하지 말고 80자 이내로 물으십시오."
    mode_instruction = "이전 답변 요약을 생략하고, '자기소개서에서 ~한 가치관이 인상적이었습니다'와 같이 자연스럽게 시작하십시오."
```

### **[4] 부정 답변(무지/회피) 대응 로직 강화**
지원자가 "모르겠습니다"라고 답변할 경우, AI가 가짜 근거를 만들어 요약(Hallucination)하지 않도록 요약을 100% 생략하고 즉시 화제를 전환하게 합니다.

**[적용 코드]**
```python
if any(kw in u_text for kw in negative_keywords):
    mode_task_instruction = "지원자가 답변을 못했습니다. 이전 답변에 대한 요약, 인용을 100% 생략하고 즉시 화제를 전환하십시오."
    global_constraint = "이전 답변 요약을 **절대** 하지 마십시오. 즉시 다른 관점에서 질문을 새로 시작하십시오."
```

## 3. 기대 효과 및 결과 분석

1.  **가독성 향상**: 질문 길이가 30% 짧아짐에 따라 구어체 특유의 늘어지는 느낌이 사라지고 면접의 템포가 빨라졌습니다.
2.  **직무-인성 분리**: 인성 면접에서 "React 구조" 같은 기술 용어가 나오지 않게 되어, 지원자의 '태도'와 '가치관'에만 온전히 집중할 수 있습니다.
3.  **할루시네이션 방지**: 답변을 모를 때 억지로 요약하지 않고 깔끔하게 화제를 전환함으로써 AI의 신뢰성을 높였습니다.
