### 1. 파싱 모델 비교 (PDF → Markdown)

표가 없는 이력서의 경우, 레이아웃을 얼마나 빠르게 파악하여 읽기 순서를 맞추느냐가 핵심입니다.

| **모델명**       | **정확도 (레이아웃/텍스트)** | **속도 (TPS/Page)** | **온프레미스 적합성** | **주요 특징**                                              |
| ---------------------- | ---------------------------------- | ------------------------- | --------------------------- | ---------------------------------------------------------------- |
| **Marker**       | **상**                       | **최상**            | 매우 높음 (가벼움)          | 텍스트 위주 문서에서 압도적인 속도. 2단 구성 파악 능력이 뛰어남. |
| **Docling**      | **최상**                     | 중상                      | 높음                        | IBM 제조. 표가 없어도 섹션 구분(Header) 능력이 가장 안정적임.    |
| **Unstructured** | 중상                               | 중                        | 보통 (Docker 권장)          | 범용성은 좋으나 로컬 설치 및 리소스 최적화가 까다로운 편.        |
| **Nougat**       | 최상                               | 저                        | 낮음 (GPU 과소비)           | 수식/논문용. 일반 이력서에는 오버스펙이며 속도가 매우 느림.      |


### 2. 정보 추출 모델 비교 (Markdown → JSON)

마크다운을 읽어 직무, 기술, 프로젝트 등을 정해진 JSON 형식으로 뽑아주는 단계입니다.

| **모델명**             | **JSON 추출 정확도** | **추론 속도 (8B 기준)** | **리소스 요구량 (VRAM)** | **주요 특징**                                                     |
| ---------------------------- | -------------------------- | ----------------------------- | ------------------------------ | ----------------------------------------------------------------------- |
| **Qwen 2.5 (7B)**      | **최상**             | **최상**                | 약 5~8GB                       | **추천.**지시 이행 능력이 소형 모델 중 1위. JSON 포맷 오류가 거의 없음. |
| **Llama 3.1 (8B)**     | 상                         | 상                            | 약 5~8GB                       | 범용성이 매우 좋으나, 복잡한 JSON 스키마에서 간혹 누락 발생.            |
| **Gemma 2 (9B)**       | 상                         | 중상                          | 약 8~10GB                      | 한국어 이해도가 높으나 Qwen 대비 JSON 출력 안정성은 소폭 낮음.          |
| **Mistral Nemo (12B)** | 상                         | 중                            | 약 12~14GB                     | 컨텍스트 윈도우가 커서 매우 긴 자기소개서 처리에 유리함.                |
