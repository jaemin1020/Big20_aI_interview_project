C. 에이전트 판단 규칙 (현재 구현 상태 및 제안)
현재 코드(
evaluator.py
, 
question_generator.py
)에는 아주 단순한 규칙만 있거나 구체적인 임계값(Threshold) 로직이 없습니다. 이를 명확히 정의해 드립니다.

답변 길이 개입?
현재: STT는 실시간으로 흐르지만, VAD(음성 감지)가 침묵을 감지하면 답이 끝난 것으로 간주합니다.
정의 (SRS용): "3초 이상 침묵 시 답변 종료로 간주" (가장 일반적인 VAD 세팅)
점수 몇 점 이하면 꼬리질문?
현재: 
question_generator.py
는 이전 대화를 참고하지만, 명시적인 점수 조건은 없습니다.
정의 (SRS용): "직전 답변의 기술 점수(technical_score)가 3점(5점 만점) 미만일 경우, 동일 토픽에 대한 심화/반론 질문(꼬리질문) 생성."
감정이 '긴장' 상태면 행동 변화?
현재: 감정 분석(deepface) 결과는 저장만 되고 질문 생성에 반영되지 않습니다.
정의 (SRS용): "최근 3회 감정 분석 결과 'Fear(긴장/두려움)' 비중이 60% 이상일 경우, 다음 질문 난이도를 1단계 하향 조정(Easy)하거나 아이스브레이킹 질문 생성."
D. 화면 단위 기준 (현재 구현 기반)
프론트엔드(frontend 폴더 및 media-server 로직 기준)를 보면 다음과 같이 화면이 구성되어 있습니다.

홈/대기 화면 (/)
목적: 사용자 로그인, 마이크/카메라 권한 승인, 면접 시작 버튼.
기능: 장치 점검, 면접 주의사항 안내.
실시간 면접 화면 (/interview)
목적: 실제 면접 진행. (가장 중요)
기능:
내 얼굴(Local Video) 및 오디오 파형 표시.
AI 면접관의 질문(텍스트/음성) 출력.
실시간 STT 자막 표시.
결과 리포트 화면 (/report)
목적: 면접 결과 확인 및 피드백.
기능: 종합 점수, 항목별 차트(기술/소통/적합성), 상세 피드백, 감정 변화 그래프.
E. 실패/예외 케이스 (현재 설계 기반)
안정성 확보를 위해 이미 고려되어 있거나 반드시 정의해야 할 예외 처리입니다.

STT 실패/침묵 시
상황: 사용자가 아무 말도 안 하거나 마이크 오류.
대응: "5초간 입력 없으면 '잘 못 들었습니다. 다시 말씀해 주시겠습니까?' 라는 기본 오디오 재생" (또는 텍스트 프롬프트 표시)
LLM 응답 지연 시 (5초 이상)
상황: GPU 과부하로 질문 생성이 늦어짐.
대응: "Fall-back Question(기본 질문) 즉시 투입."
아까 보셨던 
get_fallback_question
 함수에 있는 "지원 동기는 무엇인가요?" 같은 미리 준비된 질문을 0.1초 만에 던져서 정적을 없앱니다.
영상 끊김 (WebRTC Error)
상황: 네트워크 불안정으로 비디오 트랙 손실.
대응: "음성 전용 모드(Audio Only)로 자동 전환 후, 사용자에게 '네트워크가 불안정하여 음성 면접으로 진행합니다' 알림 표시."
📋 현재 시스템의 주요 State 정의 (분석 결과)
시스템에는 크게 3가지 핵심 State가 존재합니다.

1. 면접 세션 상태 (Interviews.status)
면접의 진행 단계를 관리하는 가장 중요한 상태입니다.

SCHEDULED: 면접 생성 직후. 아직 시작 안 함. (대기 화면)
IN_PROGRESS (또는 LIVE): 사용자가 입장하여 면접 진행 중.
COMPLETED: 모든 질문/답변 종료. 리포트 생성 대기 중.
ANALYZED: 비동기 분석(Celery) 완료. 리포트 열람 가능.
CANCELLED: 면접 중도 포기 또는 에러.
2. 사용자 연결 상태 (WebSocket/WebRTC)
실시간 통신의 연결 상태입니다. (Redis 등에서 관리 필요)

CONNECTING: 시그널링 중 (ICE Candidate 교환).
CONNECTED: WebRTC/WS 연결 완료. 미디어 스트리밍 중.
DISCONNECTED: 네트워크 끊김. (재연결 시도 필요)
FAILED: 연결 완전 실패. (오디오 전용 모드 전환 등 예외처리 필요)
3. AI 에이전트 상태 (FSM 관점)
면접관(AI)이 현재 무엇을 하고 있는지에 대한 상태입니다.

IDLE: 대기 중 (아무 말 안 함).
LISTENING: 사용자 답변 듣는 중 (STT 수신).
THINKING: 답변 분석 및 다음 질문 생성 중 (LLM 추론).
SPEAKING: 질문 발화 중 (TTS 재생).
