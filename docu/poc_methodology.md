# PoC 검증 방법 및 절차 (Detailed Methodology)

본 장은 생성형 AI 기반 멀티모달 면접 서비스의 PoC(Proof of Concept) 검증을 위해
**모든 모델(LLM / STT / 영상 분석)에 공통 적용한 검증 방법, 절차, 평가지표 산출 방식**을 상세히 기술한다.
본 검증은 단순 성능 비교가 아닌 **실제 서비스 적용 가능성 검증**을 목적으로 한다.

---

## 1. 전체 검증 프로세스 개요

PoC 검증은 다음의 **4단계 파이프라인**으로 수행하였다.

① 검증 데이터 준비
→ ② 통제 환경 구축
→ ③ 시나리오 기반 실행
→ ④ 지표 측정 및 평가

- 모든 후보 모델은 **동일 데이터, 동일 환경, 동일 절차**에서 검증
- 결과 재현성과 객관성을 확보하는 것을 최우선 원칙으로 함

---

## 2. 단계별 검증 방법 (How – Step by Step)

---

## 2.1 검증 데이터 준비 (Data Preparation)

### 목적

- 데이터 차이에 따른 성능 왜곡 방지
- 실제 면접 환경을 최대한 재현

### 데이터 구성

| 구분   | 구성                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------ |
| 텍스트 | 직군별 PDF 이력서 15종                                                                           |
| 음성   | 평균30초, 최대1분30초 실시간발화                                                                 |
| 영상   | 표정 변화가 명확한 1분 내외 면접 영상(분노,불쾌,불안,기쁨,슬픔,놀람,중립)7가지 감정을 연기한 1분 |

### 방법

- 모든 모델에 **동일 입력 데이터**를 순차 투입
- 데이터 무작위 변경 없음 (재현성 확보 목적)

---

## 2.2 통제 환경 구축 (Controlled Environment)

### 목적

- 하드웨어·환경 변수 제거
- 모델 자체 성능만 비교

### 환경 통제 항목

| 항목      | 통제 방법                            |
| --------- | ------------------------------------ |
| GPU       | 동일 GPU 사용, VRAM 할당 고정        |
| CPU / RAM | 동일 서버 환경                       |
| 네트워크  | 온프레미스 모델 외부 통신 차단       |
| 실행 방식 | 단일 모델 실행 → 멀티모달 병렬 실행 |

---

## 2.3 시나리오 기반 실행 (Scenario-based Test)

### 목적

- 단순 정확도가 아닌 **실제 서비스 흐름 검증**

### 시나리오 유형

| 유형          | 설명                        |
| ------------- | --------------------------- |
| 정상 시나리오 | 질문 → 답변 → 꼬리 질문   |
| 단답 시나리오 | “네”, “잘 모르겠습니다” |
| 무관 답변     | 질문과 무관한 발화          |
| 장문 답변     | 1분 이상 연속 발화          |
| 예외 상황     | 잡음, 표정 미검출           |

### 실행 방식

- 각 시나리오를 **모델별 5회 반복 실행**
- 중간 오류 발생 시에도 흐름 유지 여부 확인

---

## 2.4 반복 추론 및 안정성 검증 (Reproducibility)

### 목적

- 일회성 성능 우수 모델 배제
- 실서비스 안정성 검증

### 방법

- 동일 입력 데이터 5회 반복 추론
- 결과 편차(Variance), 지연 시간 분산, 오류 발생 여부 측정

---

# 3. 평가지표 및 측정 방법 (Metrics & Measurement)

---

## 3.1 정량 지표 (Quantitative Metrics)

---

### ① 정확도 (Accuracy)

#### 측정 대상

- STT: 음성 → 텍스트 변환
- LLM: 핵심 키워드 포함 여부
- 영상: 감정 분류 결과

#### 측정 방법

- **STT**
  WER = (치환 + 삭제 + 삽입) / 전체 단어 수
  CER = (글자 오류 수) / 전체 글자 수
- **LLM**
  Accuracy = (GT 키워드 포함 응답 수) / (전체 질문 수)
- **영상**
  Accuracy = (정답 감정 분류 프레임 수) / (전체 프레임 수)

---

### ② 지연 시간 (Latency)

#### 정의

- TTCT (Time To Completion of Turn)

#### 측정 방법

Latency = 결과 출력 시각 - 입력 종료 시각

- 음성: 발화 종료 → 텍스트 출력
- LLM: 질문 입력 → 답변 시작
- 영상: 프레임 입력 → 감정 결과 반환
- 평균값 + 최대값 동시 기록

---

### ③ 에이전트 완주율 (Success Rate)

#### 정의

Success Rate = (면접 종료까지 도달한 세션 수) / (전체 세션 수)

- 중간 오류, 무한 루프, 상태 꼬임 발생 시 실패 처리

---

### ④ 자원 효율성 (Resource Usage)

#### 측정 항목

- GPU VRAM 최대/평균 점유율
- 멀티모달 동시 실행 시 안정성

#### 측정 방법

- nvidia-smi 로그 수집
- 시나리오 실행 전·중·후 비교

---

## 3.2 정성 지표 (Qualitative Metrics)

---

### ① 문맥 유지력

| 기준                | 평가         |
| ------------------- | ------------ |
| 이전 답변 반영 여부 | 상 / 중 / 하 |
| 꼬리 질문 적절성    | 상 / 중 / 하 |

---

### ② 페르소나 일관성

| 기준              | 평가                    |
| ----------------- | ----------------------- |
| 면접관 톤 유지    | 유지 / 일부 이탈 / 붕괴 |
| 반말·비전문 표현 | 없음 / 일부 / 빈번      |

---

### ③ 환각(Hallucination) 발생 여부

| 기준                    | 평가               |
| ----------------------- | ------------------ |
| 이력서 미기재 내용 생성 | 없음 / 경미 / 심각 |
| 사실 왜곡               | 없음 / 있음        |

---

# 4. 평가 기준 및 최종 판단 기준 (Decision Rule)

### 점수화 예시 (5점 척도)

| 항목       | 가중치 |
| ---------- | ------ |
| 정확도     | 30%    |
| 지연 시간  | 25%    |
| 안정성     | 20%    |
| 자원 효율  | 15%    |
| 보안·운영 | 10%    |

### 최종 채택 기준

- 총점 80점 이상
- Latency 목표치 충족
- 에이전트 완주율 ≥ 95%
- 데이터 보안 요건 충족

---

## 5. 검증 과정 요약

> 본 PoC 검증은 단일 성능 비교가 아니라,
> 실제 면접 환경을 재현한 통제 조건에서
> 정확도·지연시간·안정성·운영 적합성을 단계적으로 검증하는 것을 목표로 수행하였다.
