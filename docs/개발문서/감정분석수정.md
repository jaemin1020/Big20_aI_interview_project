# 감정 분석 및 오디오 자신감 점수 수정 내역

본 문서는 면접 데이터 저장 오류 및 오디오 점수 감도 문제를 해결한 내역을 정리합니다.

## 1. DB 저장 오류 수정 (이중 JSON 인코딩)
- **문제**: `transcripts` 테이블의 `emotion` 컬럼(JSONB)에 이미 문자열화된 JSON이 저장되어 이스케이프 문자(`\`)와 중복 따옴표가 발생하는 현상.
- **수정**: `backend-core/routes/interviews.py`에서 `json.dumps()` 과정을 제거하고 딕셔너리 객체를 직접 전달하도록 변경.

## 2. 데이터 덮어쓰기 방지
- **문제**: AI 평가기(`evaluator.py`)가 결과 저장 시 행동 분석 데이터(감정, 시선 등)를 `"neutral"`과 같은 임시 값으로 덮어쓰는 현상.
- **수정**:
  - `ai-worker/db.py`: `update_transcript_sentiment`가 데이터가 있는 필드만 업데이트하도록 개선.
  - `ai-worker/tasks/evaluator.py`: 평가 단계에서 기존 행동 분석 데이터를 보존하기 위해 감정 업데이트 호출 제거.

## 3. 오디오 자신감 점수 감도 개선
- **문제**: 목소리 크기나 자신감에 관계없이 `audio` 점수가 항상 90점대이거나 변별력이 부족한 현상.
- **수정**: `media-server/main.py`
  - **로그(dB) 스케일링**: 선형 RMS 대신 데시벨 기반 계산 방식을 도입하여 사람의 청각 특성을 반영.
  - **동적 범위 확대**: 아주 작은 소리(`-40dB`)부터 충분히 큰 소리(`-15dB`)까지 점수대를 넓게 분포(30~100점).
  - **발화 지속성 강화**: 단순히 큰 소리를 내는 것뿐만 아니라, 문장을 적절히 이어가는지(발화 비율)에 대한 기준 강화.

## 4. 비전 점수(시선, 미소, 자세) 기준점 및 범위 조정
- **문제**: 무표정이나 일반적인 상태일 때의 점수 기준이 모호하고, 변별력이 부족함.
- **수정**: `media-server/main.py`
  - **보통/무표정 기준점 (Neutral)**: 아무것도 하지 않은 평범한 상태일 때 **60점**이 나오도록 고정.
  - **감점/가점 체계**: 
    - **자신감**: 단순히 미소가 없는 것(60점)과 찡그리는 것(20점)을 구분하기 위해 '찡그림(Frown)' 지표 추가.
    - **범위**: 최하 **20점**에서 최상 **100점**까지 역동적으로 변화.
  - **수식어**: `점수 = (수치 * 0.8) + 20` (단, 미소는 찡그림과 합산하여 60점 중심 계산)

## 5. 수정 파일 목록
- `backend-core/routes/interviews.py` (DB 저장 로직)
- `ai-worker/db.py` (데이터 보호 로직)
- `ai-worker/tasks/evaluator.py` (덮어쓰기 방지)
- `media-server/main.py` (오디오/비전 점수 알고리즘 고도화)
- `media-server/vision_analyzer.py` (찡그림 분석 추가)

## 6. 최종 검증 결과 요약
- **데이터 무결성**: DB에 이스케이프 문자 없이 깨끗한 JSON으로 저장됨.
- **오디오 감도**: 목소리 크기에 따라 30점대에서 90점대까지 민감하게 반응.
- **비전 감도**: 
    - **무표정/보통 태도**: 약 **60점** 내외 기록.
    - **미소/정면 응시**: **80~100점**으로 가점.
    - **찡그림/시선 회피**: **20~40점**으로 확실한 감점.

> [!NOTE]
> **순서 및 점수 관련 참고 사항 (수정 완료)**
> 1. **점수 오버플로 수정**: 정서 점수가 100점을 초과하던 버그를 수정했습니다.
> 2. **데이터 순서 보존**: 
>    - `backend-core/db_models.py`에서 컬럼 타입을 `JSONB` -> `JSON`으로 변경하였습니다.
>    - 미디어 서버에서 `OrderedDict`를 사용하여 전송 순서를 고정하였습니다.
>    - 이제 DB 저장 및 조회 시 사용자님이 요청하신 `q_idx`, `gaze`, `audio`, ... 순서가 그대로 유지됩니다.

## 7. PostgreSQL Enum 대소문자 및 매칭 로직 고도화 (최종 수정)
- **문제**: 
    - PostgreSQL `speaker` Enum이 대문자(`AI`, `USER`)로 정의되어 있으나, 코드에서 `"User"`(소문자 혼용)를 사용하여 쿼리 결과가 0건으로 조회됨.
    - 단순히 리스트 인덱스로 매칭하던 방식이 실제 데이터 순서와 어긋나 점수가 누락되는 문제.
- **수정**:
    - **Enum 일치**: `db_models.py`와 프론트엔드(`App.jsx`)의 발화자 식별자를 `"USER"` 대문자로 통일하여 DB와 완벽히 동기화.
    - **정밀 매칭 도입**: `q_idx`가 어떤 `question_id`인지 AI 발화 기록을 통해 정확히 식별하고, 해당 ID를 가진 사용자 답변 기록을 1:1로 매칭하여 업데이트.
    - **대소문자 방어**: `interviews.py` 로직에서 안전하게 매칭되도록 보강.
- **결과**: `sentiment_score`와 `emotion` 데이터가 요청하신 JSON 형식(`q_idx`, `gaze`, `audio`, `smile`, `posture`, `emotion`, `total`, `frames`)으로 정확히 저장됨.
