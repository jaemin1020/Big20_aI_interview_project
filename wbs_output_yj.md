# 산출물1 : Final_Project_모델_설계서

---

## 1. 문서 개요

### 1.1 목적
본 문서는 생성형 AI 기반 멀티모달 면접 서비스에 적용되는  
LLM, 영상 분석, 음성 인식(STT) 모델의 후보군을 정의하고,  
비교 기준, 검증 방법, 검증 데이터 선정 기준을 설계하는 것을 목적으로 한다.

---

## 2. 온프레미스 기반 LLM 설계

### 2.1 LLM 선정 배경
- 외부 API 의존 최소화
- 개인정보 보호 및 데이터 통제
- 응답 지연 최소화

### 2.2 LLM 후보 모델
- Solar-10.7B
- Llama-3.1-8B
- 기타 경량 LLM(비교군)

### 2.3 LLM 비교 기준
- 응답 정확도
- 지시사항 준수 능력
- 프롬프트 일관성
- 추론 속도
- GPU 메모리 사용량

---

## 3. 영상 분석 모델 설계

### 3.1 적용 목적
- 면접 중 표정 기반 감정 분석
- 비언어적 반응 분석

### 3.2 영상 분석 후보 모델
- DeepFace
- FER 기반 CNN 모델
- Transformer 기반 감정 인식 모델

### 3.3 비교 기준
- 감정 분류 정확도
- 실시간 처리 가능 여부
- 프레임 처리 속도
- 조명/각도 변화에 대한 강건성

---

## 4. STT 모델 설계

### 4.1 STT 후보 모델
- Nova-1
- Nova-2

### 4.2 비교 기준
- 한국어 인식 정확도
- 지연 시간
- 잡음 환경 대응력
- 장문 발화 처리 안정성

---

## 5. 검증 방법 설계

### 5.1 공통 검증 절차
1. 동일 입력 데이터 적용
2. 동일 환경에서 추론 수행
3. 정량/정성 지표 수집
4. 결과 비교 분석

### 5.2 검증 지표
- 정확도
- 처리 시간
- 실패율
- 리소스 사용량

---

## 6. 검증 데이터 선정 기준

### 6.1 데이터 구성
- 실제 면접 발화 데이터
- 표정/음성 포함 영상 데이터
- 다양한 발화 길이 및 난이도

### 6.2 데이터 선정 원칙
- 직무/인성 질문 균형
- 다양한 감정 상태 포함
- 잡음/비정형 데이터 포함

---

## 7. 기대 효과
- 모델 선정 근거 명확화
- 시스템 성능 예측 가능
- 운영 환경에 적합한 모델 도출

# 산출물3 : Final_Project_감정분석_설계서

---

## 1. 문서 개요

### 1.1 목적
본 문서는 면접 중 수집되는 음성 및 영상 데이터를 기반으로  
감정 분석을 수행하는 워커(Worker) 구조와 처리 절차를 설계한다.

---

## 2. 감정분석 워커 개요

### 2.1 역할
- 음성 및 영상 데이터 수신
- 전처리 수행
- 감정 추론 실행
- 결과 정규화 및 집계
- DB 저장 연동

---

## 3. 입력 데이터 설계

### 3.1 입력 형태
- 음성 파일 경로
- 영상 파일 경로
- 면접 세션 ID

---

## 4. 전처리 설계

### 4.1 음성 전처리
- 노이즈 제거
- 음성 구간 분리
- 샘플링 정규화

### 4.2 영상 전처리
- 프레임 추출
- 얼굴 검출
- 품질 낮은 프레임 제거

---

## 5. 감정 추론 설계

### 5.1 추론 모델
- 영상 감정 분석: DeepFace
- 음성 감정 분석: 음성 특징 기반 모델

### 5.2 감정 카테고리
- neutral
- positive
- negative
- stress

---

## 6. 결과 정규화 및 집계

### 6.1 정규화 기준
- 모델별 출력 스케일 통일
- 확률 기반 점수화

### 6.2 집계 방식
- 시간 단위 평균
- 면접 세션 단위 종합 점수

---

## 7. DB 저장 연동

### 7.1 저장 항목
- session_id
- emotion_type
- emotion_score
- timestamp

---

## 8. 실패 및 재시도 처리

### 8.1 실패 유형
- 파일 누락
- 추론 실패
- 모델 응답 오류

### 8.2 처리 기준
- 재시도 횟수 제한
- 실패 로그 기록
- 상태(State) 반영

---

## 9. 기대 효과
- 비언어적 면접 평가 가능
- 평가 신뢰도 향상
- 면접 결과 분석 고도화

# 산출물4 : Final_Project_UI/UX_화면설계서

# with : 이승재
---

## 1. 문서 개요

### 1.1 목적
본 문서는 AI 기반 면접 서비스의 사용자 화면(UI) 및 사용자 경험(UX) 설계를 정의한다.  
면접 진행 흐름과 상태(State)를 직관적으로 전달하는 화면 구성을 목표로 한다.

---

## 2. 면접 진행 화면 구성
## 시나리오별 메인 화면 구성 및 단계별, 상황별 화면 구성

### 2.1 주요 화면
- 면접 시작 화면
- 질문 제시 화면
- 답변 입력 화면(음성/영상/텍스트)
- 평가 대기 화면
- 결과 요약 화면

---

## 3. 실시간 상태 표시

### 3.1 상태 표시 요소
- 현재 면접 단계(Stage)
- 진행 상태(진행 중 / 평가 중)
- 재시도 여부
- 남은 질문 수

---

## 4. 화면 전환 로직 연동

### 4.1 전환 기준
- State 변경 이벤트 기반 화면 전환
- next_action 값에 따른 자동 이동

---

## 5. 오류 및 예외 UI

### 5.1 오류 유형
- STT 실패
- LLM 응답 오류
- 네트워크 오류

### 5.2 UI 처리 기준
- 사용자 안내 메시지 제공
- 재시도 버튼 제공
- 심각 오류 시 세션 종료 안내

---

## 6. UX 설계 기준
- 면접 흐름 방해 최소화
- 불필요한 입력 최소화
- 피드백 명확성 확보

---

## 7. 기대 효과
- 사용자 몰입도 향상
- 면접 진행 안정성 확보
- 오류 상황 대응력 강화
