# 산출물1 : Final_Project_DB_연동_기준서

---

## 1. 문서 개요

### 1.1 목적

본 문서는 시스템 전반의 데이터 흐름과 DB 연동 기준을 정의한다.
면접 서비스에서 발생하는 상태(State) 중심 데이터의 저장, 갱신, 활용 기준을 명확히 하여
안정적인 서비스 운영과 확장성을 확보하는 것을 목표로 한다.

---

## 2. 데이터 흐름 설계

### 2.1 전체 데이터 흐름

- 사용자 입력(음성/영상/텍스트)
- STT / 영상 분석 처리
- 질문 생성 및 답변 평가(LLM)
- 상태(State) 갱신
- DB 저장 및 조회
- 결과 리포트 제공

---

## 3. 데이터 저장소 설계

### 3.1 저장소 구성

- RDBMS: 사용자, 면접, 질문, 답변, 평가, 상태 데이터
- 로그 저장소: API 요청/응답, 오류 로그
- 벡터 DB: RAG용 임베딩 데이터

---

## 4. 핵심 데이터 구조 정의

### 4.1 핵심 엔티티

- USER
- INTERVIEW_SESSION
- QUESTION
- ANSWER
- EVALUATION
- STATE_HISTORY

---

## 5. 상태(State) 데이터 설계

### 5.1 State 구성 요소

- scenario
- stage
- intent
- evaluation
- next_action
- retry_count
- history

### 5.2 State 저장 기준

- 질문/답변 턴 단위 저장
- 상태 변경 시 이력 기록

---

## 6. DB 연동 기준 정의

### 6.1 연동 원칙

- API 단위 트랜잭션 처리
- 상태 갱신과 데이터 저장의 원자성 보장
- 실패 시 Rollback 처리

---

## 7. 데이터 활용 기준

### 7.1 활용 범위

- 실시간 화면 표시
- 후속 질문 판단
- 평가 결과 리포트 생성
- 서비스 개선 및 로그 분석

---

## 8. 기대 효과

- 데이터 흐름 일관성 확보
- 상태 기반 서비스 안정성 강화
- DB 구조 확장 용이

# 산출물2 : Final_Project_UI/UX_화면설계서

# with : 조윤재

---

## 1. 문서 개요

### 1.1 목적

본 문서는 AI 기반 면접 서비스의 사용자 화면(UI) 및 사용자 경험(UX) 설계를 정의한다.
면접 진행 흐름과 상태(State)를 직관적으로 전달하는 화면 구성을 목표로 한다.

---

## 2. 면접 진행 화면 구성

### 2.1 주요 화면

- 면접 시작 화면
- 질문 제시 화면
- 답변 입력 화면(음성/영상/텍스트)
- 평가 대기 화면
- 결과 요약 화면

---

## 3. 실시간 상태 표시

### 3.1 상태 표시 요소

- 현재 면접 단계(Stage)
- 진행 상태(진행 중 / 평가 중)
- 재시도 여부
- 남은 질문 수

---

## 4. 화면 전환 로직 연동

### 4.1 전환 기준

- State 변경 이벤트 기반 화면 전환
- next_action 값에 따른 자동 이동

---

## 5. 오류 및 예외 UI

### 5.1 오류 유형

- STT 실패
- LLM 응답 오류
- 네트워크 오류

### 5.2 UI 처리 기준

- 사용자 안내 메시지 제공
- 재시도 버튼 제공
- 심각 오류 시 세션 종료 안내

---

## 6. UX 설계 기준

- 면접 흐름 방해 최소화
- 불필요한 입력 최소화
- 피드백 명확성 확보

---

## 7. 기대 효과

- 사용자 몰입도 향상
- 면접 진행 안정성 확보
- 오류 상황 대응력 강화

---

## 1. 문서 개요

### 1.1 목적

본 문서는 면접 서비스에서 사용되는 API의 기능 단위 정의와
요청/응답 구조 및 상태(State) 기반 설계를 명확히 한다.

---

## 2. API 설계 원칙

- 기능 단위 API 분리
- 화면 이벤트 기반 호출
- 상태(State) 중심 설계
- 비동기 처리 우선 적용

---

## 3. 주요 API 기능 정의

### 3.1 세션 관리 API

- 면접 세션 생성
- 세션 상태 조회
- 세션 종료

### 3.2 질문/답변 API

- 질문 조회
- 답변 전송
- 후속 질문 요청

### 3.3 평가 API

- 답변 평가 요청
- 평가 결과 조회

---

## 4. Request / Response 스키마

### 4.1 Request 주요 항목

- session_id
- user_input
- state

### 4.2 Response 주요 항목

- next_question
- evaluation
- next_action
- state

---

## 5. 비동기 처리 API 설계

- STT 처리 비동기화
- 영상 분석 비동기화
- LLM 평가 비동기 처리

---

## 6. 오류 및 예외 응답 설계

### 6.1 오류 응답 구조

- error_code
- error_message
- retry_available

---

## 7. 기대 효과

- API 역할 명확화
- 프론트엔드 연동 용이
- 서비스 확장성 확보
