## 1. 개요
인터뷰 지연 현상 및 분석 품질 저하 문제를 해결하기 위한 분석 결과 및 상세 실행 계획입니다.

## 2. 문제 분석 요약
- **지연 시간**: 프론트엔드의 5분 강제 폴링 루프 이슈.
- **분석 품질**: CPU 워커 실행 시 분석을 건너뛰는 "FAST MODE" 로직 및 범용 프롬프트 사용 이슈.
- **성능 저하**: 방대한 전체 로그를 한꺼번에 분석하는 비효율적인 리포트 생성 방식.

## 3. 상세 해결 계획 (Roadmap)

### 3.1 [지연 시간 개선] 프론트엔드-백엔드 동기화
- **App.jsx**: 질문 폴링 시 서버로부터 `is_finished` 신호를 확인하도록 수정. 종료 신호 수신 시 즉시 인터뷰 마무리 로직 실행.
- **AI Worker**: 시나리오상의 최대 질문 도달 시 인터뷰 종료 상태를 DB에 반영하는 로직 추가.

### 3.2 [품질 개선] 분석 엔진 고도화 및 품질 보장
- **evaluator.py (실시간 분석)**: 
    - "FAST MODE"(분석 생략) 로직을 제거하여 모든 답변에 대해 LLM 분석 보장.
    - 지원자의 '지원 직무(Position)' 정보를 프롬프트에 주입하여 전문성 있는 평가 수행.
- **evaluator.py (리포트 생성)**: 
    - 전체 로그 요약 방식에서 '중간 평가 데이터 취합' 방식으로 최적화하여 연산 속도 개선.

### 3.3 [정확도 개선] 데이터 정합성 강화
- 마지막 답변이 DB에 완전히 기록된 것을 확인한 후 리포트 생성을 시작하도록 비동기 타이밍 보정.

## 4. 기대 효과
- **UX 개선**: 마지막 질문 후 대기 시간을 5분에서 즉시 또는 수초 내외로 단축.
- **품질 향상**: 직무 맞춤형 분석 및 모든 답변에 대한 실시간 피드백 데이터 확보.
- **안정성**: 동시 접속자가 많아도 분석 데이터 누락 없는 리포트 생성.
