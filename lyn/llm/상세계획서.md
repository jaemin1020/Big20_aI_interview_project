AI 모의면접 프로그램의 '심장'과 '두뇌'에 해당하는 가장 핵심적인 세 분야를 맡으셨군요! 이 기술들은 단순히 답변을 생성하는 수준을 넘어, **"면접관처럼 생각하고, 정확한 근거로 평가하는 능력"**을 부여하는 역할입니다.

각 분야가 왜 중요한지, 그리고 어떤 기능을 하는지 비유를 곁들여 상세히 설명해 드릴게요.

---

## 1. VectorDB 설계: "면접관의 전문 지식 도서관"

VectorDB는 텍스트를 숫자의 배열(벡터)로 변환해 저장하는 특수 저장소입니다.

* **왜 쓰이나?** : AI가 모든 전공 지식이나 기업 정보를 암기할 수는 없습니다. 방대한 데이터 중에서 지금 면접 질문에 딱 맞는 정보를 **초고속으로 찾아내기 위해** 사용합니다.
* **비유** : 면접관 옆에 놓인 **'수만 권의 전공 서적과 면접 족보가 꽂힌 인덱스 도서관'**과 같습니다. 질문이 들어오면 해당 키워드와 맥락이 가장 유사한 페이지를 순식간에 펼쳐주는 역할입니다.

| **주요 기능**             | **상세 설명**                                                                                                                |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **벡터화 및 임베딩 전략** | 문장을 컴퓨터가 이해하는 좌표값으로 바꿉니다. 단어의 뜻뿐만 아니라 '문맥'을 이해하게 만드는 핵심 과정입니다.                       |
| **메타데이터 설계**       | 데이터에 '태그'를 다는 작업입니다. (예: 삼성전자 관련, 신입용, 직무기술서 등) 이를 통해 필요한 정보만 필터링해서 찾을 수 있습니다. |
| **검색 방식 정의**        | 단순히 단어가 겹치는 게 아니라, 의미가 얼마나 유사한지(Cosine Similarity 등) 계산하여 최적의 참고 자료를 뽑아냅니다.               |

## 2. RAG(검색 증강 생성) 설계: "면접관의 논리적 사고 체계"

RAG는 LLM이 가진 일반적인 지식에 VectorDB에서 찾은 '특수 지식'을 결합하는 전략 설계입니다.

* **왜 쓰이나?** : AI가 지셔내지 않은 거짓말을 하는 '환각(Hallucination)' 현상을 막기 위해서입니다. "우리 회사의 인재상이 뭐야?"라고 물었을 때, 엉뚱한 소리가 아니라 **실제 회사의 문서에 기반한 정확한 질문과 평가**를 하기 위해 필수적입니다.
* **비유** : 면접관이 질문하기 전, **'지원자의 자소서와 회사의 채용 공고문을 읽고 질문 리스트를 정리하는 사고 과정'**입니다. 근거(Context)를 가지고 논리적으로 말하게 만드는 규칙입니다.

| **주요 기능**           | **상세 설명**                                                                                                                                  |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Chunk 전략**          | 긴 문서를 AI가 읽기 좋게 적절한 크기로 자르는 기술입니다. 너무 길면 핵심을 놓치고, 너무 짧으면 맥락이 끊깁니다.                                      |
| **프롬프트 주입 구조**  | "너는 면접관이야. 아래 전달해주는 [지원자 자소서] 내용을 바탕으로 꼬리 질문을 던져줘"라고 명령 체계를 만드는 것입니다.                               |
| **실패·Fallback 설계** | 만약 관련 정보를 못 찾았다면? 헛소리를 하는 대신 "죄송하지만 관련 정보를 찾을 수 없습니다"라고 대처하거나 일반적인 질문으로 전환하는 안전장치입니다. |

## 3. RAG/LLM 구현: "면접관의 실제 목소리와 판단"

설계도를 바탕으로 실제로 코드를 작성하여 시스템이 돌아가게 만드는 단계입니다.

* **왜 쓰이나?** : 설계된 로직이 실제 사용자(면접자)의 입력에 반응하여 실시간으로 동작하게 하기 위함입니다. LangChain 같은 프레임워크를 사용해 데이터 흐름을 연결합니다.
* **비유** : 도서관과 사고 체계를 갖춘 면접관이 **'실제로 입을 열어 질문을 던지고, 답변을 듣고 점수를 매기는 실제 행동'** 그 자체입니다.
* **이 기술이 중요한 이유** : 아무리 좋은 엔진이 있어도, 답변이 너무 늦게 나오거나(Latency), 점수 형식이 이상하면 서비스로 사용할 수 없기 때문입니다.

| **주요 기능**              | **상세 설명**                                                                                              |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **지식 적재 파이프라인**   | PDF나 텍스트 파일인 자소서/직무기술서를 자동으로 VectorDB에 집어넣는 자동화 통로를 만듭니다.                     |
| **Top-K & Threshold 설정** | 검색된 결과 중 상위 몇 개(K개)를 참고할지, 유사도가 몇 % 이상인 것만 믿을지 결정하여 답변의 정확도를 조절합니다. |
| **출력 파싱(Parsing)**     | AI가 내놓은 긴 텍스트에서 '질문 내용', '평가 점수', '피드백'만 쏙쏙 뽑아 화면에 예쁘게 보여주도록 정제합니다.    |
