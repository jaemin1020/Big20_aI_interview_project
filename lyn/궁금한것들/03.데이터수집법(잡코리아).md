IT·데이터 직무에 한정한 모의면접이라면, 사람인에서는 “채용공고 API + 상세 공고 페이지”에서 직무 정의에 필요한 필드를 구조화해서 가져오는 게 좋습니다.[[oapi.saramin.co](https://oapi.saramin.co.kr/guide/job-search)]

## 1. 어떤 정보를 가져오면 좋은지

특히 인터뷰 질문 생성에 바로 쓰기 좋은 정보는 다음입니다.[[law.sookmyung.ac](https://law.sookmyung.ac.kr/law/board/job.do?mode=download&articleNo=76025&attachNo=49790)]

* 기본 메타데이터
  * 공고 id, 공고 URL, 회사명, 지역, 고용형태(정규직/계약직 등)
  * 경력(min/max, 신입/경력), 학력, 연봉 범위 등.[[oapi.saramin.co](https://oapi.saramin.co.kr/guide/job-search)]
* 직무·직종 정보
  * `job-mid-code`(상위 직무), `job-code`(세부 직무), 각 코드에 대응하는 이름 (예: “백엔드 개발자”, “데이터 분석가”).[[saramin.co](https://www.saramin.co.kr/zf_user/jobs/list/job-category?cat_mcls=2)]
  * 직무 키워드 코드/텍스트(`job-code-keyword-code`, `keyword`) → 기술스택, 도메인 키워드.[[oapi.saramin.co](https://oapi.saramin.co.kr/guide/job-search)]
* 포지션 설명(핵심)
  * 공고 제목(`position.title`).
  * 상세 공고 페이지에 있는:
    * “담당업무” / “주요업무”
    * “자격요건” (필수/우대)
    * “우대사항”
    * “기술스택/툴” 섹션.[[saramin.co](https://www.saramin.co.kr/zf_user/jobs/list/job-category?cat_mcls=2)]

사람인 Job Search API 응답에는 요약 정보, 직무 코드·경력·학력·키워드가 포함되고, 상세 JD 본문은 공고 URL에 접속해서 HTML을 파싱해야 하는 구조입니다.[[oapi.saramin.co](https://oapi.saramin.co.kr/guide/job-search)]

## 2. 어떻게 수집할지 (플로우)

IT·데이터 카테고리만 타깃으로 수집하는 예시입니다.[[saramin.co](https://www.saramin.co.kr/zf_user/jobs/list/job-category?cat_mcls=2)]

1. **IT·데이터 직무 공고만 필터링**
   * 사람인 IT개발·데이터 카테고리(cat_mcls=2 등)를 기준으로 Job Search API 호출.[[saramin.co](https://www.saramin.co.kr/zf_user/jobs/list/job-category?cat_mcls=2)]
   * 또는 `job-mid-code` / `job-code`를 IT·데이터 계열 코드만 사용해서 검색.
2. **Job Search API로 1차 메타데이터 수집**
   * 각 공고에 대해 다음 필드를 저장:
     * `id`, `url`, `company.name`, `position.title`
     * `job-mid-code.code/name`, `job-code.code/name`
     * `experience-level.min/max/name`
     * `required-education-level.name`
     * `keyword`(기술 키워드 문자열) 등.[[oapi.saramin.co](https://oapi.saramin.co.kr/guide/job-search)]
   * 이건 일반 DB (예: `job_postings` 테이블)에 그대로 저장.
3. **공고 상세 페이지에서 JD 본문 파싱**
   * `url` 필드로 각 공고 HTML에 접속한 뒤:
     * “담당업무”, “자격요건”, “우대사항”, “기타” 등 섹션별 텍스트를 추출.[[law.sookmyung.ac](https://law.sookmyung.ac.kr/law/board/job.do?mode=download&articleNo=76025&attachNo=49790)]
   * 추출 결과를 `job_posting_details` 같은 테이블에:
     * `posting_id` (FK)
     * `responsibilities_text`
     * `requirements_text`
     * `preferred_text`
     * `stack_text`(있으면)

       이런 식으로 저장.
4. **RAG용 chunk 생성 & 벡터DB 저장**
   * 위 섹션 텍스트를 문단/토큰 단위로 나누어 chunk로 만들고, 각 chunk에 메타데이터를 붙임.[[unstructured](https://unstructured.io/blog/chunking-for-rag-best-practices)]
   * 예:
     * `text`: “주요 업무: 데이터 파이프라인 설계 및 ETL 구현…”
     * `metadata`:
       * `job_mid_code="IT개발·데이터"`, `job_code="데이터 엔지니어"`
       * `section="responsibilities"`
       * `experience_level="신입"`, `stack_tag=["Python","SQL","Airflow"]`(키워드에서 파싱).[[law.sookmyung.ac](https://law.sookmyung.ac.kr/law/board/job.do?mode=download&articleNo=76025&attachNo=49790)]

## 3. 모의면접에서 어떻게 활용할지

이렇게 수집·저장한 데이터를 모의면접 질문 생성에 쓰는 방법입니다.[[mercity](https://www.mercity.ai/blog-post/build-an-llm-based-resume-analyzer)]

* 지원자 측 입력
  * 이력서 텍스트 + 지원 직무(예: “데이터 분석가(주니어)”).
* RAG Retrieval
  * 벡터DB에서 `job_code="데이터 분석가"` + 유사도 검색으로:
    * 실제 JD의 “담당업무/자격요건/우대사항” chunk 가져오기.
* LLM 프롬프트에 포함
  * “다음은 한국 사람인에서 수집한 실제 데이터 분석가 공고들의 공통된 담당업무/자격요건이다…” + 지원자 이력서 요약 →

    “이 요구사항을 검증할 수 있는 기술/경험 인터뷰 질문 5개 생성” 같은 지시.[[arxiv](https://arxiv.org/html/2504.02870v1)]

이 구조를 쓰면, “사람인에 실제로 올라오는 IT·데이터 공고 기준으로 만든 모의면접”이 되기 때문에, 질문이 시장 요구와 더 잘 맞습니다.[[saramin.co](https://www.saramin.co.kr/zf_user/jobs/list/job-category?cat_mcls=2)]
